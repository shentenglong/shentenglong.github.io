VUE+springBoot项目部署

上周花了半天时间部署了一个前后端分离的项目，中间遇到了很多问题，于是整理了一下，在此记录。

VUE部署步骤（不修改前端路径）
1.下载nginx，http://nginx.org/en/download.html
2.在前端项目文件夹处打开CMD运行 npm run build 或 使用VSCODE打开前端项目在终端中运行npm run build 也可以直接执行build脚本
3.执行完成之后会生成一个dist文件夹。
4.将下载的nginx压缩包解压，复制到服务器上的某一个位置，如D:\guquan
5.打开nginx文件夹，找到html文件夹，将上面的dist文件夹放到html中，如D:\guquan\nginx-1.18.0\html
6.在nginx文件夹中打开conf，找到nginx.conf，这个是nginx的配置文件，先复制一份，防止修改出现问题无法恢复。
7.打开原生的nginx.conf,在server {}这个中修改以下内容：
(1) 修改listen 为你需要监听的端口号，注意不能是服务器已经在使用的端口号。
(2) 修改location / {}中的root为你的dists文件夹路径，默认开始路径是nginx文件夹，所以此处我们只需要填html/dist。
(3)在location/{}中加上index index.html index.htm;和 try_files $uri $uri/ /index.html;这个是为了解决VUE页面刷新报404的问题。
(4)增加后端的请求代理，在location / {}的下面增加
location /api/v2/ {
rewrite ^/api/v2/(.*)$ /$1 break;
proxy_pass http://192.168.61.68:8082/;
}
此处的api/v2在前端项目的vue.config.js中的proxy{}里面查看，不同的前端项目配置不同。如果proxy{}中配置了多个代理，则需要打开前端项目的login.js或api下的其他JS文件查看具体的请求格式。proxy_pass后面是后端部署的服务器以及端口号。
修改完成之后大概是这样的格式

8.保存修改的nginx.conf文件，在nginx文件夹路径启动CMD，或者CD进入nginx文件夹路径。然后输入 start nginx；服务就开始启动了。如果在nginx服务启动之后需要修改配置文件，那么修改完成之后在CMD中输入 nginx -s reload 就可以刷新服务，不需要重启。所以nginx启动之后的CMD窗口不要关闭。
9.上述步骤完成之后，在浏览器输入你前端部署的服务器加上监听的端口号。比如我前端部署在192.168.61.68这台服务器上。那么我输入192.168.61.68:8099/就可以访问前端页面了；如果这个时候无法访问页面就需要排查错误：
(1)输入192.168.61.68:8099/之后直接报500错误，检查root后面的dist文件夹路径是否正确。
(2)页面可以正常打开但是点击登录提示网络错误或找不到该资源，打开前端项目的vue.config.js，查看dev或devServe中的port是否是前面nginx.conf中监听的端口号，host是否是’0.0.0.0’,外层的publicPath是否为’/’，都检查完成之后重新启动运行 npm run build,将生成的dist覆盖之前的。再次启动nginx就可以正常访问页面了。
