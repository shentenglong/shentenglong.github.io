VUE和前后端分离

六月初我被分到一个项目组，参与开发一个前后端分离的项目，因为人手不够，所以我既要写后端接口又要写前端页面。可问题在于我之前完全没有使用过VUE
，连基础的语法都不是很清楚。没办法只能硬着头皮上，白天啃一啃别人写好的项目，晚上回去看一看VUE的视频。就这样慢慢的也能写写页面，现在过去了一个多月,我
对于VUE和前后端问题也有了一点自己的想法，在此记录一下。

以前我只是看过别人写好的前后端项目，对于前后端分离这个概念一直没有一个准确的理解。现在自己上手做了之后，才觉得前后端这个思想的出现是必然，前段和后端的代码
不可能永远放在一起，不仅影响开发的效率，也不利于后面的维护。前段和后端本质上只存在数据交互，所以为什么不把它们分开呢？前端只需要专注于页面的样式
和布局，数据完全可以用假数据代替，后端把接口完成之后给到前端，一个页面就完成了。在这种想法下,VUE出现了。它和springBoot的完美配合就是我对于前后端分离
项目的最初印象。所以接下来就聊一聊VUE；

对于一个接触VUE不到两个月的人来说，VUE最让我感到神奇的地方就是它的组件。你可以在开发项目前写好多个公共组件，比如表单，详情页。这些公共组件
支持你在任何页面调用，就如同JAVA中封装好的方法一般，即是一种规范，也是提升效率的有效途径。其次就是它的一些指令，比如v-model，这个双向绑定解决了很多数据交互方面的问题。尤其是
VUE还为一些常用指令提供了缩写，非常方便！

以上只是我个人对于VUE和前后端分离的简单看法，由于我对此接触较浅，如有错误，还请指正。后续随着我个人能力的提升，也会继续更新这篇文章，毕竟学习是一个长久的过程，需要不断的记录！

VUE+springBoot项目部署

上周花了半天时间部署了一个前后端分离的项目，中间遇到了很多问题，于是整理了一下，在此记录。

VUE部署步骤（不修改前端路径）
1.下载nginx，http://nginx.org/en/download.html
2.在前端项目文件夹处打开CMD运行 npm run build 或 使用VSCODE打开前端项目在终端中运行npm run build 也可以直接执行build脚本
3.执行完成之后会生成一个dist文件夹。
4.将下载的nginx压缩包解压，复制到服务器上的某一个位置，如D:\guquan
5.打开nginx文件夹，找到html文件夹，将上面的dist文件夹放到html中，如D:\guquan\nginx-1.18.0\html
6.在nginx文件夹中打开conf，找到nginx.conf，这个是nginx的配置文件，先复制一份，防止修改出现问题无法恢复。
7.打开原生的nginx.conf,在server {}这个中修改以下内容：
(1) 修改listen 为你需要监听的端口号，注意不能是服务器已经在使用的端口号。
(2) 修改location / {}中的root为你的dists文件夹路径，默认开始路径是nginx文件夹，所以此处我们只需要填html/dist。
(3)在location/{}中加上index index.html index.htm;和 try_files $uri $uri/ /index.html;这个是为了解决VUE页面刷新报404的问题。
(4)增加后端的请求代理，在location / {}的下面增加
location /api/v2/ {
rewrite ^/api/v2/(.*)$ /$1 break;
proxy_pass http://192.168.61.68:8082/;
}
此处的api/v2在前端项目的vue.config.js中的proxy{}里面查看，不同的前端项目配置不同。如果proxy{}中配置了多个代理，则需要打开前端项目的login.js或api下的其他JS文件查看具体的请求格式。proxy_pass后面是后端部署的服务器以及端口号。
修改完成之后大概是这样的格式

8.保存修改的nginx.conf文件，在nginx文件夹路径启动CMD，或者CD进入nginx文件夹路径。然后输入 start nginx；服务就开始启动了。如果在nginx服务启动之后需要修改配置文件，那么修改完成之后在CMD中输入 nginx -s reload 就可以刷新服务，不需要重启。所以nginx启动之后的CMD窗口不要关闭。
9.上述步骤完成之后，在浏览器输入你前端部署的服务器加上监听的端口号。比如我前端部署在192.168.61.68这台服务器上。那么我输入192.168.61.68:8099/就可以访问前端页面了；如果这个时候无法访问页面就需要排查错误：
(1)输入192.168.61.68:8099/之后直接报500错误，检查root后面的dist文件夹路径是否正确。
(2)页面可以正常打开但是点击登录提示网络错误或找不到该资源，打开前端项目的vue.config.js，查看dev或devServe中的port是否是前面nginx.conf中监听的端口号，host是否是’0.0.0.0’,外层的publicPath是否为’/’，都检查完成之后重新启动运行 npm run build,将生成的dist覆盖之前的。再次启动nginx就可以正常访问页面了。

Mysql与Oracle常用函数

公司的一个项目之前是使用Oracle数据库，现在需要改为Mysql,在数据库迁移工作完成之后就需要对代码中sql语句进行修改,在修改的过程中发现Mysql跟Oracle的查询函数还是有很多相同之处的，觉得需要记录一下。

1.DATE类型与VARCHAR转换
这个使用的最为频繁，因为业务表中几乎都会有create_time和update_time两个字段；
oracle：
to_char(t.PUBLISH_DATE,’yyyy-mm-dd’) PUBLISH_DATE DATE转VARCHAR
to_date(t.PUBLISH_DATE,’yyyy-mm-dd’) PUBLISH_DATE VARCHAR转DATE
mysql：
date_format(t.PUBLISH_DATE, ‘%Y-%m-%d’) PUBLISH_DATE; DATE转VARCHAR
str_to_date(t.PUBLISH_DATE, ‘%Y-%m-%d’) PUBLISH_DATE; VARCHAR转DATE
其中的转换格式都可以根据自己的需求去改变。

2.将多个字符串拼接在一起
Oracle:
‘,’||T.TYPE_ID||’,’;
mysql：
concat(‘,’,T.TYPE_ID,’,’);

3.将一列值用逗号隔开成为一条数据
oracle:
select wmsys.wm_concat(it.type_name) from GM_INFO_TYPE it;
mysql:
select group_concat(it.type_name) from GM_INFO_TYPE it;

4.对一个值进行重新赋值
oracle和mysql都是使用case when then
case
when 条件
then 条件为true,执行then后面的语句
else 条件为false，执行else后面的语句
end
如果有多个when，oracle中需要写完整的条件，而Mysql不需要，具体如下:
mysql:
case when a.type=’A’ then ‘这是A’ when ‘B’ then ‘这是B’ end typeName;
oracle:
case when a.type=’A’ then ‘这是A’ when a.type=’B’ then ‘这是B’ end typeName;
当然Mysql也支持同Oracle一样的写法，我个人也更偏向与Oracle的写法，比较完整。

5.查询每列数据的序号
Oracle中自带ROWNUM，而Mysql则需要自己构造一下
oracle:
SELECT ROWNUM AS ID from table_name;
Mysql:
SELECT @rowno:=@rowno+1 as ID from table_name r ,(select @rowno:=0) t;

6.树查询也即递归查询
Oracle中可以直接使用START WITH …CONNECT BY
Mysql中还没有支持递归查询的函数，但是可以通过构造临时表进行简单的递归查询，例如:
Oracle:
select a.id from table_name as a start with a.id = 10000011 CONNECT BY a.id =PRIOR a.pid(父级ID);
Mysql:
with recursive temp as( select a.* from table_name as a where a.id = 10000011 union all select a.* from table_name as a,temp t where t.id=a.pid)SELECT id FROM temp t;
with recursive temp as 即创建一个 temp表。

7.树查询带深度
Oracle:
select a.id,LEVEL from table_name as a start with a.id= 10000011 CONNECT BY a.id =PRIOR a.pid(父级ID);
Mysql
with recursive temp as( select a.,1 leveal from table_name as a where a.id = 10000011 union all select a.,t.leveal+1 leveal from table_name as a,temp t where t.id=a.pid)SELECT t.id,t.leveal FROM temp t;

//
// 示例 1：
//
// 输入： 2

//

初入算法：每日温度题

由于一直听别人说到过算法，自己从网上搜也没有得到想要的答案。于是就在IDEA上安装了一个Leetcode插件，工作之余偶尔刷刷题目。说是刷题，其实也就是按照我自己的想法来写，还是没有感受到什么是算法。直到今天碰到了一道题目，让我第一次感受到算法的作用。所以特意记录一下。
题目如下：
//请根据每日 气温 列表，重新生成一个列表。对应位置的输出为：要想观测到更高的气温，至少需要等待的天数。如果气温在这之后都不会升高，请在该位置用 0 来代替。
//
//
// 例如，给定一个列表 temperatures = [73, 74, 75, 71, 69, 72, 76, 73]，你的输出应该是 [1, 1, 4, 2
//, 1, 1, 0, 0]。
//
// 提示：气温 列表长度的范围是 [1, 30000]。每个气温的值的均为华氏度，都是在 [30, 100] 范围内的整数。
// Related Topics 栈 哈希表
刚看到这个题目感觉很简单，只需要循环比较就可以得到答案，没觉得跟算法有什么关系。正当我提交完答案准备下一题的时候看到了一个不一样的解法。
简单来说，我的做法是从左往右进行循环，每计算一个都需要全部遍历。很浪费时间。那么假设我在计算75对应位置的输出时，知道了71对应位置的值为2，那么我就可以直接用75去跟72比较，不需要与69进行比较。这样在数组长度较长的时候可以极大的缩短代码运行的时间。
